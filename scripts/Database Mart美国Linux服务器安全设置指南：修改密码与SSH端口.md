## 一、修改用户密码详细步骤

1. **登录服务器**
   - 使用SSH工具连接服务器
   - 执行命令获取root权限：
     bash
     sudo -i
     

2. **修改密码**
   - 修改指定用户密码：
     bash
     passwd 用户名
     
   - 示例（修改root密码）：
     bash
     passwd root
     
   - 系统会提示输入并确认新密码

3. **验证修改**
   - 断开当前SSH连接
   - 使用新密码重新登录验证

## 二、修改SSH端口完整流程

### 准备工作
- 确保已通过SSH登录服务器
- 建议先保持当前SSH会话，同时开启新终端测试新端口

### 操作步骤

1. **编辑配置文件**
   bash
   vi /etc/ssh/sshd_config
   

2. **修改端口参数**
   - 找到`#Port 22`行
   - 取消注释（删除#）
   - 修改为1024-65535之间的端口号，例如：
     
     Port 2222
     

3. **保存并重启服务**
   - 保存修改：按`ESC`后输入`:wq`
   - 重启SSH服务：
     bash
     service sshd restart
     

4. **防火墙设置**
   - 如果启用了防火墙，需放行新端口
   - 以UFW为例：
     bash
     ufw allow 2222/tcp
     

## 安全建议

1. 定期更换密码（建议每3个月一次）
2. 避免使用常见端口号
3. 考虑启用SSH密钥认证
4. 修改端口后，建议禁用root直接登录
